Home Lab Overview

This setup is a self-hosted home lab: a private computing environment running on personal hardware. It is used to host services, experiment with infrastructure, and maintain full control over data and system configuration.

The core of the lab consists of virtualized servers running multiple applications inside isolated network segments (VLANs). Instead of exposing these services directly to the internet, access is handled through a secure relay architecture.

Because typical residential connections are behind CGNAT, direct inbound access is not possible. To solve this, the lab establishes an outbound encrypted tunnel to a public cloud instance. This cloud node acts as a controlled entry point, forwarding requests into the internal network without exposing it.

The result is a system that combines privacy, control, and real-world infrastructure design, while remaining accessible from anywhere through a secure and managed path.

How it Works

  • Client Request: A request is sent over the internet and resolved via DNS to a public cloud endpoint.
  • Cloud Entry Point: The request reaches a cloud-hosted instance acting as the public ingress.
  • Reverse Proxy: A proxy receives the request and forwards it instead of handling it locally.
  • Secure Tunnel: Traffic is routed through a WireGuard tunnel initiated from inside the private network.
  • Internal Routing: The request enters the internal VLAN and is directed to the correct service.
  • Processing: Backend services on virtualized servers handle the request.
  • Response: The response returns through the same path back to the client.

Key Characteristics

  • No direct exposure of internal services
  • Single controlled entry point via cloud
  • Bypasses CGNAT using outbound tunnel
  • Supports fallback cloud nodes
